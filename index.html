代码驿站用户使用说明

1. 联系我们

我们的后台网站在这里: https://codexexp.com/

购买后建议先加入售后QQ群。群内会同步重要通知、维护公告、配置更新，也可以咨询使用问题。

如需要人工协助，也可以在群里添加客服QQ。

遇到问题时，请尽量提供以下信息：

  • 注册邮箱
  • 订单号或兑换码截图
  • 报错截图
  • 使用的客户端，例如 Codex CLI、VSCode 插件

2. 安装 Codex 客户端

如果您之前已经用过Codex，本地环境已经安装好了，那么可以直接看第3步。

如果您使用第三方客户端，只需要从我们这里获得一个 api 地址，那么可以直接看第3步。

如果您第一次使用Codex，那么可以继续看本节。我们推荐新用户优先使用 Codex app；专业技术人员也可以使用 VSCode 插件；习惯命令行的用户可以使用 Codex CLI

2.1 安装 Codex app（推荐）

  1. Codex app 是 OpenAI 开发的官方桌面软件，很好用，可以在这里下载 https://openai.com/zh-Hans-CN/codex/
  2. 下载正常安装即可，如果Windows用户遇到问题的话，也可以自行前往 微软商店 搜索 Codex 下载安装。
  3. 安装好之后不要登录账号，也不要输入api key。

2.2 安装 VSCode 插件（推荐）

如果不需要使用VSCode的话，则不用安装。

  1. 安装并打开 VSCode，下载链接 https://code.visualstudio.com/。
  2. 点击左侧“扩展”图标。
  3. 搜索 Codex。
  4. 找到 OpenAI Codex 插件，点击安装。
  5. 安装完成后，重启 VSCode。

2.3 安装 Codex CLI

Codex CLI 需要先安装 Node.js。

  1. 打开 Node.js 官网 https://nodejs.org/en/download，下载 Windows 安装包。
  2. 双击安装包，按提示一路 Next。
  3. 安装完成后，重新打开终端。
  4. 在终端输入以下命令，确认安装成功：
node -v
npm -v

如果能显示版本号，说明 Node.js 已安装成功。如果windows上提示权限不足等问题，可以尝试执行：
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

然后安装 Codex CLI：

npm install -g @openai/codex

如果提示权限不足，请右键“以管理员身份运行”终端后再试。如果你在使用Codex的时候频繁出现中文乱码的问题，推荐将本地PowerShell升级到版本 7，可以解决乱码问题。

3. 兑换码使用指南

兑换码区分大小写，请完整复制，不要带前后空格。

3.1 新客户

  1. 打开代码驿站官网。
  2. 点击“登录/注册”。
  3. 选择“注册新账号”，填写邮箱和密码。
  4. 登录后，进入左侧“兑换”页面。
  5. 粘贴兑换码，点击“兑换”或“确认”。
  6. 可以看到余额的变化，或者当前订阅的套餐变化。

3.2 老客户

  1. 直接登录已有账号。
  2. 进入左侧“兑换”页面。
  3. 粘贴新的兑换码并提交。
  4. 可以看到余额的变化，或者当前订阅的套餐变化。

注意：如果你之前已经创建过 API 密钥，兑换后通常不用重新创建。若更换了套餐分组，请检查密钥是否选择了正确分组。

4. API 密钥使用教程

API 密钥是连接 Codex、VSCode 插件或其他客户端的凭证。不要把密钥发给无关人员。

4.1 创建 API 密钥

  1. 登录代码驿站。
  2. 点击左侧“API 密钥”。
  3. 点击“创建密钥”。
  4. 填写名称，例如“apikey”。
  5. 在“分组”里选择你已兑换的套餐。
  6. 点击“创建”。

💡 小贴士：及时切换到月卡分组
如果你是这样的使用路径：

  1. 先充值了 次卡（例如 10 美刀余额）
  2. 体验后觉得不错，果断加购了 月卡
    ⚠️ 那么一定要记得 —— 及时切换到月卡分组。
    否则次卡余额消费完了，在 Codex 或 CLI 上会提示 余额不足。

4.2 获取客户端配置

  1. 在 API 密钥列表中找到刚创建的密钥。
  2. 点击“使用密钥”。
  3. 选择你要使用的客户端，例如 Codex CLI 或 Codex VSCode 插件。
  4. 按页面提示复制配置，或下载配置脚本。
  5. 配置完成后，重新打开终端或重启 VSCode。

⭐ 强烈推荐：一键配置

打开页面后直接使用「一键配置」，三步搞定。

📌 第一步：退出 Codex App

  • ✅ 退出当前登录账号
  • ✅ 完全关闭 Codex App

📌 第二步：执行对应系统脚本

  • 💻 Windows
    下载左侧脚本 → 双击运行
  • 🍎 macOS
    复制右侧命令 → 打开「终端」→ ⌘ + V → 回车

📌 第三步：重新打开 Codex App

✨ 完成！

🛠️ DIY 用户：手动配置

手动修改 base_url、API Key 等配置项。

📂 配置文件位置

  • 🪟 Windows
    Win + R → 输入 %userprofile%\.codex
  • 🍏 macOS / Linux
    路径：~/.codex

⚠️ 注意

  • 若目录不存在，请先手动创建
  • 配置完成后，务必重启客户端生效

5. 常见问题 QA

Q1：兑换成功后，为什么余额没有增加？

订阅套餐不一定会增加账户余额。很多套餐是增加“订阅额度”，使用 API 时消耗订阅额度。请到“我的订阅”查看套餐是否有效。

Q2：兑换码提示失败怎么办？

请检查兑换码是否完整、大小写是否正确、是否带了空格、是否已经使用过。仍然失败时，请联系售后并提供注册邮箱、订单号和兑换码截图。

Q3：API 密钥创建后还能再次查看完整密钥吗？

通常创建后应立即复制保存。若密钥丢失，可以删除旧密钥后重新创建一个新的。

Q4：配置后 Codex 还是不能用怎么办？

请按顺序检查：

  1. 是否已经兑换成功。
  2. “我的订阅”是否显示有效，当前余额是否大于0。
  3. API 密钥是否选择了正确分组。
  4. 是否复制了完整配置。
  5. 是否重启了终端、Codex CLI 或 VSCode。

Q5：Codex 里看不到某个模型怎么办？

部分客户端界面可能不会显示所有模型。实际调用模型以配置文件和网站使用记录为准。修改配置后，请重启客户端。

Q6：可以把同一个 API 密钥放到多台电脑上吗？

可以，但不建议随意分享。多人共用同一个密钥时，用量会算在同一个账号下，也不方便排查问题。

Q7：我应该用 VSCode 插件还是 Codex CLI？

不熟悉命令行的用户建议先用 VSCode 插件；需要在终端里操作项目、执行命令的用户可以用 Codex CLI。
